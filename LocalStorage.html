LocalStorage

¿Qué es el IndexedDB?


Con el LocalStorage o HTML5 Web Storage podemos guardar información en nuestro navegador web a modo de sesión y que esa información persista y esté disponible durante la navegación entre las diferentes páginas de nuestro sitio o aplicación web.

El LocalStorage suele usarse mucho en aplicaciones web desarrolladas completamente con JavaScript, con tecnologías como Angular, aunque también puede aplicarse a cualquier web en la cual necesitemos compartir datos entre secciones.

En las aplicaciones web monolíticas desarrolladas con un lenguaje de backend como PHP o cualquier otro, el uso del LocalStorage es sustitudo por las Sesiones del propio lenguaje de backend ya que el frontend y el backend están completamente integrados y mezclados.

Si desea almacenar objetos o matrices, deberá convertirlos en cadenas mediante el método JSON.stringify(). Puede almacenar hasta 5 MB de datos en localStorage. Además, todas las ventanas con el mismo origen pueden compartir los datos de almacenamiento local de ese sitio.




¿Cómo funciona LocalStorage?

Puede acceder a la funcionalidad localStorage a través de la propiedad Window.localStorage. Esta propiedad proporciona varios métodos como setItem(), getItem() y removeItem(). Puede usarlos para almacenar, leer y eliminar pares clave/valor.

Puede almacenar datos en localStorage utilizando el método setItem(). Este método acepta dos argumentos, la clave y el valor correspondiente.

window.localStorage.setItem('Python', 'Guido van Rossum');

Aquí, Python es la clave y Guido van Rossum es el valor. Si desea almacenar una matriz o un objeto, deberá convertirlo en una cadena. Puede convertir una matriz o un objeto en una cadena mediante el método JSON.stringify().


Características de LocalStorage


➤ Permite almacenar entre 5 MB y 10 MB de información, esto incluye texto y multimedia.

➤ La información está almacenada en la computadora del cliente y No es enviada en cada petición del servidor, a diferente de las cookies.

➤ Se utiliza un número mínimo de peticiones al servidor, todo esto para reducir el tráfico de red que se puede generar.

➤ Previene perdidas de información cuando se desconecta de la red.




Comandería de LocalStorage


➤ Validar objeto Storage en el navegador


Esto sirve para los navegadores que no son compatibles, para que nos demos cuenta en que navegadores no se puede utilizar el Storage, aunque en la actualidad la mayoría de navegadores son compatibles pero no esta demás hacer está validación.

Código:

if (typeof(Storage) !== 'undefined') {

// Código cuando Storage es compatible

} else {

// Código cuando Storage NO es compatible

}


➤ Guardar Datos en LocalStorage


Para guardar datos, se puede realizar de dos maneras:

Código:

// Opción 1 -> localStorage.setItem(name, content)

// Opción 2 ->localStorage.name = content

localStorage.setItem('Nombre', 'Cristian Alexander')

localStorage.Apellido = 'Santos Escobar'

Nuestro name, es el nombre del elemento y el content es igual a nuestro contenido del elemento.

➤ Recuperar datos en LocalStorage

Al igual que para agregar información, para recuperarla existen diferentes maneras de hacerlo:

Código:

// Opción 1 -> localStorage.getItem(name, content)

// Opción 2 -> localStorage.name

// Obtenemos los datos y los almacenamos en variables

let firstName = localStorage.getItem('Nombre'),

lastName = localStorage.Apellido

console.log(`Hola, mi nombre es ${firstName} ${lastName}`)

//Esto nos mostrará: Hola, mi nombre es Cristian Alexander Santos Escobar

➤ Eliminar datos de LocalStorage

Para eliminar un elemento dentro de LocalStorage haremos lo siguiente:

Código:

// localStorage.removeItem(name)

localStorage.removeItem(Apellido)

➤ Limpiar todo el Storage

Ya para finalizar veremos la forma para eliminar todos los datos del Storage y dejarlo completamente limpio

Código:

localStorage.clear()
