SessionStorage

¿Qué es el SessionStorage?


SessionStorage permite guardar información en un navegador, similar a una cookie a diferencia que SessionStorage permite guardar la información solo cuando la ventana de ejecucion este abierta ya que cuando se cierra la información almacenada se pierde.

Esto tambien nos lleva a la diferencia de localStorage y SessionStorage, en localStorage nosotros podemos guardar información sin que se pierda es decir que nosotros podemos cerrar la ventana o navegador y al momento de ejecutarla de nuevo la información siempre estaría guardada, eso no pasa en SessionStorage ya que al momento de cerrar el navegador o la ventana la información ya no se guardara.

¿Cómo funciona el SessionStorage?


SessionStorage se utiliza como objeto global de javaScript que disponen métodos que permite manipular datos que se quieran utilizar. La SessionStorage guardar información/datos que serán accesibles únicamente al navegador con el que se inicio la conexión. Cabe aclarar que en SessionStorage si se abre otra ventana de ejecución tendrá un almacenamiento distinto y SessionStorage soporta el refresco de la página sin que se pierdan los datos.

Características de SessionStorage


➤ Permite almacenar entre 5 MB y 10 MB de información, esto incluye texto y multimedia.

➤ La información está almacenada en la computadora del cliente y No es enviada en cada petición del servidor, a diferente de las cookies.

➤ Se utiliza un número mínimo de peticiones al servidor, todo esto para reducir el tráfico de red que se puede generar.

➤ Previene perdidas de información cuando se desconecta de la red.

Métodos y propiedades de SessionStorage


➤ setItem(clave, valor) – almacenar un par clave/valor

➤ getItem(clave) – obtener el valor por medio de la clave

➤ removeItem(clave) – eliminar la clave y su valor

➤ clear() – borrar todo

➤ key(índice) – obtener la clave de una posición dada

➤ length – el número de ítems almacenados

Para ver los datos que estamos almacenando simplemente damos clic derecho en nuestra página web, luego damos clic en inspeccionar y nos vamos a la parte de "Application" y seleccionamos SessionStorage para ver lo que estamos guardando.

Comandería de SessionStorage


Validar objeto Storage en el navegador

La validación siempre es esencial en los dos casos, siempre es bueno validarlo por si se presenta un caso en el que nuestro navegador no sea compatible.

Código:

if (typeof(Storage) !== 'undefined') {

// Código cuando Storage es compatible

} else {

// Código cuando Storage NO es compatible

}


Guardar Datos en SessionStorage

Para guardar datos, se hace de la siguiente manera:

Código:

// Opción 1 -> sessionStorage.setItem(name, content)

// Opción 2 ->sessionStorage.name = content

sessionStorage.setItem('Nombre', 'Cristian Alexander')

sessionStorage.Apellido = 'Santos Escobar'

Nuestro name, es el nombre del elemento y el content es igual a nuestro contenido del elemento.

Recuperar datos en SessionStorage

Al igual que para agregar información, para recuperarla existen diferentes maneras de hacerlo:

Código:

// Opción 1 -> sessionStorage.getItem(name, content)

// Opción 2 -> sessionStorage.name

// Obtenemos los datos y los almacenamos en variables

let firstName = sessionStorage.getItem('Nombre'),

lastName = sessionStorage.Apellido

console.log(`Hola, mi nombre es ${firstName} ${lastName}`)

//Esto nos mostrará: Hola, mi nombre es Cristian Alexander Santos Escobar

Eliminar datos de SessionStorage

Para eliminar un elemento dentro de sessionStorage haremos lo siguiente:

Código:

// sessionStorage.removeItem(name)

sessionStorage.removeItem(Apellido)

Limpiar todo el Storage

Ya para finalizar veremos la forma para eliminar todos los datos del Storage y dejarlo completamente limpio

Código:

sessionStorage.clear()
